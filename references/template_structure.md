# 影刀 RPA 需求模板文档结构

## 模板文件

标准模板已内置在 skill 中：

`C:/Users/13251/.workbuddy/skills/video-to-rpa-doc/references/影刀RPA需求文档模板.docx`

该模板基于 `生产入库单自动写入金蝶对应字段-需求详情表(脱敏但不可外发).docx` 同款样式重建，特点：
- 2 节 A4 页面（21cm × 29.7cm），所有页面带统一页边框
- 封面使用浮动文本框（WPS textbox）：标题、副标题、版本+日期共 3 个文本框；影刀RPA logo 为浮动形状内嵌
- 脚本通过 lxml 直接操作 WPS 文本框 XML，替换占位文字
- 封面还有 1 个装饰线条形状，保持不变
- 保留 3 个表格：流程基本信息、应用系统、详细步骤解析（后追加）
- 表格占位文字已清理，可直接作为模板使用

## 页面节 (Sections)

模板有 2 个节（封面 + 正文），页面大小为 A4（约 11906 × 16838 EMU），带四边页边框（single, sz=4, auto）。

## 封面文本框

封面页包含 3 个浮动 WPS 文本框的占位文字，生成脚本会按内容匹配替换：

| 文本框 | 占位文字 | 用途 |
|--------|---------|------|
| 1 | `流程名称` | 替换为实际标题 |
| 2 | `流程简介 / 副标题` | 替换为副标题（为空时清空） |
| 3 (P0) | `Version 1.0` | 替换为版本号 |
| 3 (P1) | `2026.06.28` | 替换为日期 |

此外还有 1 个浮动形状（影刀RPA logo）和 1 个装饰线，脚本不修改。

## 表格结构

### Table 0: 流程基本信息 (2行 × 6列)

| 列 | 字段 |
|----|------|
| 0 | 流程名称 |
| 1 | 流程所在部门 |
| 2 | 流程主题场景描述 |
| 3 | 单次操作时间 |
| 4 | 每天重复量（次） |
| 5 | 备注/说明 |

- Row 0: 表头
- Row 1: 数据行（替换填充）

### Table 1: 应用系统 (5行 × 7列)

| 列 | 字段 |
|----|------|
| 0 | 系统名称 |
| 1 | 内部/外部 |
| 2 | 系统类型 (Windows/Web/iOS等) |
| 3 | 是否有密码 |
| 4 | 是否有验证码 |
| 5 | 最近有无升级计划 |
| 6 | 备注/说明 |

- Row 0: 表头
- Row 1~4: 数据行（模板有4行占位，实际数据行数可变）
- 生成脚本会删除多余空行，保留 表头 + 实际数据行数

### Table 2: 详细步骤解析 (21行 × 3列)

| 列 | 字段 |
|----|------|
| 0 | 步骤号 |
| 1 | 步骤名称 + 详细说明 + 截图 |
| 2 | 其他补充说明及资料 |

每个步骤占 2 行：
- **奇数行 (1, 3, 5, ...)**: 步骤号 + 步骤名称
- **偶数行 (2, 4, 6, ...)**: 详细说明文字 + 截图 + 补充说明

模板有 10 个步骤占位（21行 = 表头 + 10步×2行），生成脚本会删除多余行。

## 关键技术点

### Word XML 行删除
Word 表格的 XML 子元素包含 `tblPr` 和 `tblGrid`，不能直接用索引删除子元素。
必须用 `tbl_xml.findall(qn('w:tr'))` 获取真正的 `<w:tr>` 行元素列表，再删除。

### 图片插入
图片插入到步骤详细说明单元格中，居中对齐，宽度通过 `Cm(width_cm)` 控制。
推荐宽度: 4~5.5cm（根据截图宽高比调整）。
